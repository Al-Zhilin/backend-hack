# 🗺️ Transit & Tour Master API

Сервер-оркестратор для планирования городских путешествий. Он связывает нейросетевой ИИ, картографические сервисы (2ГИС, Яндекс) и облачную базу данных MongoDB Atlas.

## 🏗️ Архитектура системы

Проект реализован как **асинхронный хаб**, где данные передаются между компонентами по принципу "состояние в базе":
1. **Frontend ↔️ ИИ**: Живой диалог для уточнения предпочтений.
2. **ИИ ➡️ Backend**: Передача "сырых" тегов для подбора мест.
3. **Backend ↔️ Frontend**: Предоставление вариантов на выбор и выдача финального маршрута.

---

## 🔄 Логика работы (Workflow)

Процесс разбит на **3 ключевых этапа**:

### Этап 1: Сбор тегов и поиск вариантов (Suggestions)
1. Пользователь общается с ИИ через роут `/api/chat` **Streaming** методом.
2. Когда ИИ собрал данные, он **сам** отправляет JSON на роут `/api/receive_tour_data`.
3. Сервер в фоне ищет реальные места через **2GIS** и сохраняет их в коллекцию `suggestions` в MongoDB.

### Этап 2: Выбор пользователя
1. Фронтенд запрашивает найденные варианты через `/api/get_suggestions/{login}`.
2. Пользователь видит карточки мест и выбирает те, которые ему нравятся.
3. Фронтенд отправляет выбранные объекты на `/api/generate_route`.

### Этап 3: Финализация и логистика
1. Сервер получает выбранные места, решает **задачу коммивояжера** (кратчайший путь) и запрашивает расписания транспорта в **Яндексе**.
2. Результат сохраняется в коллекцию `generated_tours`.
3. Фронтенд забирает готовый план через `/api/get_user_route/{login}` и рисует его на карте.

## 🛠️ Описание Роутов

### 🤖 Для Нейросети
*   **`POST /api/chat`**: Прокси-чат. Фронтенд отправляет текст, получает поток (stream) от ИИ.
*   **`POST /api/receive_tour_data`**: Прием структурированных тегов от ИИ. Запускает фоновый поиск мест.

### 📱 Для Frontend-разработчика
*   **`GET /api/get_suggestions/{login}`**: Получение списка найденных карточек мест.
    *   *Если 404:* Места еще ищутся (покажите спиннер).
    *   *Если 200:* Отрисуйте массив `suggested_places`.
*   **`POST /api/generate_route`**: Отправка массива выбранных пользователем мест. Сервер ответит "OK" и начнет строить маршрут в фоне.
*   **`GET /api/get_user_route/{login}`**: Получение финального маршрута.
    *   *Результат:* Отсортированные шаги (`route_steps`) и расписание транспорта (`schedule`).

## 🛠 Технологический стек

*   **FastAPI**: Высокопроизводительный асинхронный фреймворк.
*   **Motor (MongoDB)**: Асинхронное взаимодействие с облачной БД Atlas.
*   **HTTPX**: Клиент для асинхронных запросов к внешним API.
*   **GIS Services**: 
    *   *Yandex Geocoder*: Преобразование адресов в координаты.
    *   *2GIS Catalog*: Поиск реальных заведений.
    *   *2GIS Routing*: Оптимизация порядка посещения точек.
    *   *Yandex Расписания*: Время работы и маршруты транспорта.

## 🚀 Быстрый старт (Разработка)

### 1. Переменные окружения (`.env`)
Создайте файл `.env` в корне:
```env
YANDEX_GEOCODER_API_KEY=...
YANDEX_RASP_API_KEY=...
DGis_API_KEY=...
MONGO_URI=mongodb+srv://...
DB_NAME=TourProject
BASE_OF_LOCAL_URL=http://localhost:5000
```

### 2. Запуск
```bash
pip install -r requirements.txt
python server.py
```
Документация будет доступна по адресу: `http://localhost:8000/docs`

## ⚠️ Важные примечания
*   **Polling (Опрос):** Поскольку генерация маршрута и поиск мест занимают время (внешние API), фронтенду необходимо опрашивать GET-роуты с интервалом 2-3 секунды, пока не придет статус 200.
*   **CORS:** На сервере разрешены все источники (`allow_origins=["*"]`), что позволяет фронтенду подключаться без дополнительных настроек.
