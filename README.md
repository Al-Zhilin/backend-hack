# 🗺️ Transit & Tour Master API (Backend)

Серверная часть приложения для умного планирования туристических маршрутов. 
Сервер выступает в роли "оркестратора": он объединяет нейросетевой ИИ, картографические данные 2GIS (поиск мест, задача коммивояжера) и сервисы Яндекса (геокодирование, расписание транспорта), чтобы создать для пользователя идеальный план прогулки.

---

## ⚙️ Архитектура ("Черный ящик")

Для внешнего наблюдателя сервер работает как конвейер из двух этапов (Human-in-the-loop):
1. **Сбор данных:** Принимает пожелания пользователя (из них нейромодель делает теги) -> превращает адреса в координаты (Яндекс) -> находит подходящие заведения в радиусе (2GIS) -> отдает их пользователю на выбор.
2. **Оптимизация (Фоновая задача):** Принимает выбранные места -> строит кратчайший маршрут между ними (Матрица 2GIS) -> собирает расписание автобусов/электричек между точками (Яндекс) -> сохраняет готовый тур в MongoDB Atlas.

Все внешние запросы (к API карт и БД) выполняются **асинхронно** (через `httpx` и `motor`), что позволяет серверу на базе FastAPI выдерживать высокие нагрузки, не блокируя основной поток.

## 📱 Руководство для Frontend-разработчика

Взаимодействие с сервером строится на логичном пошаговом флоу (Workflow).

### Базовый URL
Если сервер запущен локально: `http://localhost:8000`
Интерактивная документация (Swagger): `/docs`

### Рабочий процесс (Workflow)

#### Шаг 0. Общение с ИИ
Пользователь пишет текстом, чего он хочет.
* **POST** `/api/chat`
* **Особенность:** Возвращает ответ через `StreamingResponse`. Фронтенду нужно читать поток данных (chunks) и печатать их на экране эффектом пишущей машинки.

#### Шаг 1. Поиск предложений по тегам
Когда система определила теги из чата, запрашиваем список мест.
* **POST** `/api/search_places`
* **Body (JSON):**
  ```json
  {
    "login": "user123",
    "status": "completed",
    "collected_tags": {
      "start_location": "Москва, Красная площадь 1",
      "transport_mode": "пешеход",
      "categories": ["кофейня", "музей"]
    }
  }
  ```
* **Response (200 OK):** Возвращает объект `start_point` и массив `suggested_places`. Фронтенд рисует из них карточки мест с чекбоксами для выбора.

#### Шаг 2. Запуск генерации маршрута
Пользователь выбрал несколько мест (например, 3 из 10 предложенных) и нажал "Построить маршрут".
* **POST** `/api/generate_route`
* **Особенность:** Сервер отвечает *мгновенно*, а тяжелая сборка маршрута запускается в фоне.
* **Body (JSON):**
  ```json
  {
    "login": "user123",
    "transport_mode": "пешеход",
    "start_point": {
      "name": "Москва, Красная площадь 1",
      "lat": 55.753544,
      "lon": 37.621202
    },
    "selected_places": [ 
       // Массив выбранных объектов (полностью скопированных из ответа Шага 1)
       { "id": "123", "name": "Кафе", "lat": 55.7, "lon": 37.6, "address": "ул. Мира", "tag": "кофейня" }
    ]
  }
  ```
* **Response (200 OK):** `{"status": "success", "message": "Места получены..."}`. Фронтенд показывает пользователю экран загрузки (спиннер).

#### Шаг 3. Получение готового маршрута (Polling)
Пока крутится спиннер, фронтенд раз в 3-5 секунд опрашивает этот эндпоинт.
* **GET** `/api/get_user_route/{login}`
* **Response (404 Not Found):** Маршрут еще считается (продолжаем опрашивать).
* **Response (200 OK):** Маршрут готов! Возвращается финальный JSON.
  * Фронтенд отрисовывает массив `route_steps` на карте. 
  * Внутри каждого шага есть массив `schedule`. 
  * *Примечание для UI:* Если прямых автобусов нет, сервер вернет текстовое пояснение в поле `title` (например: `"🚶 Ближайших остановок не найдено..."`).

## 🛠 Технологический стек

* **Фреймворк:** [FastAPI](https://fastapi.tiangolo.com/) (Python 3.10+)
* **База данных:** MongoDB Atlas (удаленное облако)
* **Асинхронные драйверы:** `motor` (для БД), `httpx` (для HTTP-запросов)
* **Валидация данных:** `pydantic`
* **Внешние API:**
  * Yandex Geocoder API (координаты старта)
  * Yandex Расписания API (транспорт)
  * 2GIS Catalog API (поиск заведений)
  * 2GIS Routing API (задача коммивояжера / оптимизация пути)

## 🚀 Установка и запуск (Для разработчиков)

### 1. Клонирование и зависимости
```bash
git clone <url-вашего-репозитория>
cd <папка-проекта>
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```
*(Убедитесь, что установлены `dnspython` и `pymongo[srv]` для работы с MongoDB Atlas)*.

### 2. Переменные окружения (.env)
Создайте файл `.env` в корневой папке проекта со следующим содержимым:

```env
# Ключи API картографических сервисов
YANDEX_GEOCODER_API_KEY=ваш_ключ
YANDEX_RASP_API_KEY=ваш_ключ
DGis_API_KEY=ваш_ключ

# Строка подключения к MongoDB Atlas (без кавычек)
MONGO_URI=mongodb+srv://<user>:<password>@cluster0.xxxx.mongodb.net/TourProject?retryWrites=true&w=majority
DB_NAME=TourProject

# Локальный URL для проксирования чата с ИИ (по умолчанию http://localhost:5000)
BASE_OF_LOCAL_URL=http://localhost:5000
```

### 3. Запуск сервера
```bash
python server.py
```
Или через uvicorn для горячей перезагрузки при разработке:
```bash
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```

## ⚠️ Возможные ошибки и их решение

* **Ошибка 500 при старте:** Неверный `MONGO_URI`. Убедитесь, что ваш текущий IP-адрес добавлен в *Network Access (Whitelist)* в панели MongoDB Atlas (`0.0.0.0/0`).
* **Ошибка 422 (Unprocessable Entity):** Фронтенд прислал JSON с неверными ключами. Проверьте схему в `/docs` (самая частая ошибка: путаница между `suggested_places` и `selected_places`).
* **Ответ 2GIS: 403 Invalid Key:** Проверьте ключ 2GIS. Для Routing API (матрица) часто требуется отдельный доступ/ключ, отличный от ключа для Каталога.
